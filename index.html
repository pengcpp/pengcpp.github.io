git相关的一些东西

发表于 2016-07-14   |   分类于 编程

记录一些与git相关的东西

想要更换git托管，如何迁移原git仓库。

先查看remote的名字，假设你的remote是origin，用 git remote set-url 更换地址，
假如想更换到github的NewProject下：

git branch -r
git remote set-url origin git@github.com:username/NewProject.git

如果你想从别的git托管那里复制一份源代码到新的git托管服务器上的话，可以通过以下步骤来操作来保留原版本库中的所有内容：

从原地址克隆一份裸版本库，比如原本托管于github

git clone --bare git://github.com/username/project.git

然后到新的git服务器(比如gitcafe)上创建一个新项目，以镜像推送的方式上传代码到gitcafe服务器上

cd project.git
git push --mirror git@gitcafe.com/username/newproject.git

到新服务器gitcafe上找到Clone地址，直接Clone到本地就可以了

git clone git@gitcafe.com/username/newproject.git

注：裸版本库不含工作区，直接就是版本库的内容，目录名以.git做后缀

windows配置ssh访问github

发表于 2016-07-12   |   分类于 编程

使用 ssh key 验证 github 的好处就是不用每次提交代码的时候都要输入用户名和密码，省去了多次提交时候的麻烦。

创建一个 ssh key

$ cd ~/.ssh

如果没有则提示：

bash: cd: /c/Users/Administrator/.ssh: No such file or directory

可以自己创建文件夹也可以在生成时创建（.shh文件夹为cygwin默认文件夹）

生成并添加第一个ssh key

$ ssh-keygen -t rsa -C "youremail@xxx.com" -f ~/.ssh/github_blog_keys

-t 指定密钥类型，默认是 rsa ，可以省略

-C 设置注释文字

-f 指定密钥文件存储文件名，如果省略会使用默认文件名，生成 id_rsa 和 id_rsa.pub 两个秘钥文件

接着又会提示你输入两次密码（此密码是你push文件的时候要输入的密码），不输入的话在push的时候就不需要输入密码，直接提交到github上。

$ Enter passphrase (empty for no passphrase):
$ Enter same passphrase again:

接下来就会显示

Your identification has been saved in /c/Users/Administrator/.ssh/github_blog_keys.
Your public key has been saved in /c/Users/Administrator/.ssh/github_blog_keys.pub.
The key fingerprint is:
SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx youremail@xxx.com
The key's randomart image is:
+---[RSA 2048]----+
|   ..   ..+      |
| o. o. . o .     |
|o ++  o   . o    |
|.=.+..   . o .   |
|+oA=.   B   .    |
|.+*.=..          |
|.+ =o=.   C      |
|..Ao+... . .     |
|.+o=..... .      |
+----[SHA256]-----+



添加 ssh key 到 github上

登录GitHub，点击右上角账号头像Settings，点击SSH and GPG keys， 选New SSH key，可以自己定义个名字，拷贝 github_blog_keys.pub 文件的内容（可以用编辑器打开复制）到对应输入框，点击Add Key 按钮后需要再次输入GitHub登录密码来确认。



测试 ssh key

在 git bash 中输入代码

$ ssh -T git@github.com
Hi username! You've successfully authenticated, but GitHub does not provide shell access.

如果提示的username正确的话，说明你已经成功设置SSH密钥。可能会出现警告，输入yes并回车就好。如果在生成ssh文件时设置了密码此处也会提示输入。

注：这里如果在生成ssh文件时 -f 后面用了非默认的名字（github_blog_keys），测试的时候可能会遇到Permission denied (publickey)的问题。

$ ssh -T git@github.com
The authenticity of host 'github.com (192.30.253.112)' can't be established.
RSA key fingerprint is SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added 'github.com,192.30.253.112' (RSA) to the list of known hosts.
Permission denied (publickey).

这时需要将 github_blog_keys 加入到ssh中，然后再输入测试命令。

$ ssh-agent bash
$ ssh-add ~/.ssh/github_blog_keys
Identity added: /c/Users/Administrator/.ssh/github_blog_keys (/c/Users/Administrator/.ssh/github_blog_keys)
$ ssh -T git@github.com
Hi pengcpp! You've successfully authenticated, but GitHub does not provide shell access.

Hexo 是一款基于Node.js的静态博客框架，可以很方便的生成静态网页托管于github、gitcafe和Heroku，类似于jekyll。 作者是台湾的tommy351 ，支持 Markdown语法，兼容 Windows, Mac & Linux。

安装配置

需要安装 Node 和 Git(主要用到git bash)。

npm install hexo -g #安装  
npm update hexo -g #升级  
hexo init #初始化，在你指定的目录

常用命令

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo publish
hexo generate #生成静态页面至public目录
hexo server #启动服务预览（默认端口4000，http://localhost:4000/查看)
hexo deploy #部署,将.deploy目录部署到GitHub

简写： 

hexo n = hexo new 
hexo g = hexo generate 
hexo s = hexo server 
hexo d = hexo deploy

复合： 

hexo deploy -g 
hexo server -g

每次部署的步骤，可按以下三步来进行

hexo clean
hexo generate
hexo deploy

目录结构

主要的目录结构

|-- node_modules
|-- public
|-- scaffolds
|-- source
|-- themes
|-- _config.yml
|-- package.json
  • node_modules hexo插件，一些安装支持。
  • public 生成的静态网页文件夹。
  • scaffolds 模版文件夹。当你创建一篇新的文章时，hexo是根据这个目录下的文件进行构建的。
  • scripts 脚本目录，放js文件的地方。
  • source 放文章的地方，还有一些主要的资源，很重要。
  • themes 主题文件夹，每一个子目录就是一个主题。
  • _config.yml 全局配置文件，网站的很多信息都在这里配置。
  • package.json 应用数据，从它可以看出hexo版本信息，以及它所默认或者说依赖的一些组件。
